# AirFuture Maintenance Workflow

このワークフローは、プロジェクトの整合性を保ち、Vercelのデプロイエラーを防ぐために。

## 1. 事前確認
- 現在の作業ディレクトリを確認してください。
- **正会員（実行コード）は常に `04_code/` の中にあります。**
- リポジトリのルートディレクトリに直接ファイルを置かないでください。

## 2. 変更の適用
- コードを修正する際は、必ず `04_code/` 下のファイルを編集してください。
- `vercel.json` を変更する場合は、`04_code/vercel.json` を使用し、リライト先の拡張子（.js）を入れないルールを遵守してください。
- **タイムゾーンの厳守**: 日付や時刻を扱うロジックを修正・追加する場合、必ず日本時間 (JST / UTC+9) を明示的に扱うコードになっているか確認してください。UTCとの混同は、予約投稿の失敗や統計の不整合（午前中の実績消滅など）を招くため、最優先のチェック項目とします。

## 3. デプロイ後のテスト
// turbo
- 修正をプッシュした後は、必ず以下のコマンドで疎通を確認してください：
  `curl -i https://x-autoup.vercel.app/api/cron?pw=airfuture2026`
- 404エラーが出る場合は、Vercelダッシュボードの「Root Directory」が `04_code` になっているか、またはファイルがその中にないかを疑ってください。
