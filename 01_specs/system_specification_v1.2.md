# AirFuture SNS 自動運用システム 仕様書 v1.2（保存版）

## 1. システム目的
| 目的 | 詳細 |
| :--- | :--- |
| **製品認知の最大化** | 空気の悩み層（花粉・臭気・インフル等）へ継続アプローチ |
| **教育による信頼獲得** | 専門知識を分かりやすく発信 |
| **自動化による運用効率化** | 投稿生成〜分析を極力自動化 |
| **LP誘導・購入転換** | プロフィール／固定投稿経由でLPへ確実に誘導 |

**評価**: 確信度 高

## 4. 教育動線（S1〜S5ファネル設計）
| ステージ | KPI | 投稿役割 | 内容例 |
| :--- | :--- | :--- | :--- |
| **S1 共感** | 反応率・保存 | 悩みの代弁 | ペット臭・花粉あるある漫画 |
| **S2 原因理解** | 信頼 | 科学知識 | 「タバコ臭は壁に付着し空気へ再拡散」図解 |
| **S3 解決方向性** | 行動喚起 | 空気ケアTips | 今日からできる消臭3選 |

| **S5 LP導線** | クリック率 | 導入事例・FAQ | before/after、医院導入例 |

**週内バランス**:
*   S1〜S3：70%
*   S4〜S5：30%

## 6. 自動生成ワークフロー（Antigravity連携）
1.  **メモ収集**: 空気の敵／知識／事例を収集
2.  **メタ付与**: ステージ(S1〜S5)、季節タグ、悩みカテゴリを付与
3.  **AI投稿生成**: キャラ仕様＋品質制御込みで生成
4.  **トレンドワード挿入判定**: 0〜2個を自動挿入
5.  **投稿カレンダーへ自動配列**: スケジュールリング
6.  **X APIへ自動投稿**: 実行
7.  **エンゲージログ自動記録**: 結果分析

## 7. データモデル設計
### posts テーブル
*   `id`: UUID
*   `draft`: 投稿本文
*   `stage`: S1-S5
*   `season`: 季節タグ
*   `hashtags`: 使用ハッシュタグ配列
*   `CTA`: CTAタイプ
*   `status`: draft_ai / scheduled / posted / failed
*   `media_url`: 画像/動画URL

### hashtags テーブル
*   `name`: タグ名
*   `engagement_score`: 反応スコア
*   `type`: 常設 or トレンド

### logs テーブル
*   `post_id`: 投稿ID
*   `CTR`: クリック率
*   `RT`: リポスト数
*   `fav`: いいね数
*   `save`: 保存数

## 8. 品質・リスク管理
| 危険 | 制御方法 |
| :--- | :--- |
| **売り込み臭** | S1-3比率70%固定＋投稿自己評価で強度チェック |
| **医療誤認** | 「傾向」「可能性」表現＋出典提示 |
| **スパム認定** | ハッシュタグ最大2個 |
| **表現ブレ** | キャラプロンプト固定＋NGワード辞書 |

## 9. 導線設計
*   **プロフィール**: LP URL＋一文「空気の敵から守る未来技術」
*   **固定投稿**: 教育＋製品＋事例＋LPのマスター構成（スレッド）
*   **CTA文例**: 「もっと知る？→プロフィールの青いリンク」
*   **週次まとめ投稿**: 今週の知識まとめ→LPへ

## 10. トレンドワード活用モジュール
*   **取得**: X(または外部API)で日次取得
*   **フィルタ**: 空気テーマ関連のみ抽出
*   **投稿反映**: 生成時に0〜2個挿入
*   **ログ分析**: engagement_score更新
